怎么使用github+hexo搭建个人博客

2018-12-24

前期配置工作

下载并安装git

1、git下载地址

2、默认配置不要去更改里面的配置，不然会报意想不到的错误，next就可以了！

3、在git里面执行 git -v检验是否安装成功。

下载并安装node

1、node下载地址

2、默认配置不要去更改里面的配置，不然会报意想不到的错误，next就可以了！、

3、在git里面执行 node -v检验node时候安装成功。

安装cnpm

1、如果你需要安装更快一点，那cnpm是不错的选择。node里面已经自带npm也可以使用，这一步你可以忽略。

2、git Bash Here 在命令里执行

npm install -g cnpm --registry=https://registry.npm.taobao.org

3、在git里面执行 cnpm -v检验node时候安装成功。

安装hexo

1、git Bash Here 在命令里执行

cnpm install -g hexo-cli

2、命令自动执行，直到安装完成

3、安装Hexo的git插件

cnpm install hexo-deployer-git --save

4、同样命令自动执行，直到安装完成。

搭建本地博客

1、我在D盘创建一个blog的文件夹，鼠标右击选择git Bash Here 打开命令行。执行

hexo init

2、处置化完成之后，继续执行命令,知道命令执行完毕。

cnpm install

3、以上工作都完成之后，启动服务。

hexo server

4、在浏览器中打开(http://localhost:4000), 就可以访问到本地安装的hexo博客了。

Hexo部署到Github

Github配置

1、注册Github账号

2、创建Github远程仓库 点击头像前的 + 找到 【new repository】

3、新建一个仓库，名字用英文。

Github与本地仓库关联

1、在D盘建的blog文件夹右击打开命令行执行

git config --global user.name '你的名字'

git config --global user.email '你的Github邮箱地址'

2、查看本机公钥，执行下面命令行。你会得到你的公钥。

cat ~/.ssh/id_rsa.pub

3、如果没有请执行下面命令行，你会得到二个文件。id_rsa.pub 、id_rsa

ssh-keygen -t rsa -C "你的Github邮箱地址"

4、在Github到个人中心，找到【SSH and GPG keys】，然后点击 【New SSH key】 把id_rsa.pub的内容复制进到内容里面，title随便写，需要英文。

5、然后执行下面命令行，测试你的本机与远程仓库是否连接成功。若出现HI后面是你的github的账号名说明连接成功。

ssh -T git@github.com

6、_config.yml文件的配置，很重要的步骤 

title: 主题
subtitle: 副标题
description: 网站描述
keywords: 关键词
author: 作者
language: 语言
timezone: 

# URL此步骤配置错误引起页面css代码加载不了，引起整个页面css样式混乱。坑1
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: 你上次代码的地址
root: github目录的名字，
permalink: :year/:month/:day/:title/
permalink_defaults:

# Deployment 注意repo的配置，不然会报错。容易出错，坑2
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:lushunde321/lushunde321.github.io.git 
  branch: master

7、以上步骤成功之后，执行下面代码。hexo -g生成静态文件，hexo -d 部署到github的仓库

hexo clean

hexo -g

hexo -d

8、此时此刻，静态文件都已经完成，还需要一个配置。到你的代码库那边，找到setting进入，
找到【GitHub Pages】，none的下拉框，选中。然后选择 master branch。然后保存，就可以通过
你的仓库访问你的个人博客了。
